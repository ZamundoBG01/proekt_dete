=== УРОК 4: ЛОГИЧЕСКИ УСЛОВИЯ И СРАВНЕНИЯ В АСЕМБЛЕР (CMP, JMP, JE) ===

1. СРАВНЕНИЕ В ПРОЦЕСОРА (CMP):
   За да вземе решение, процесорът използва инструкцията CMP (Compare).
   Пример: CMP RAX, 100 (Процесорът сравнява стойността в RAX с числото 100).

2. УСЛОВНИ ПРЕХОДИ (JUMPS):
   След CMP процесорът проверява резултата и може да "прескочи" към друг участък от кода:
   - JE (Jump if Equal): Прескочи, ако стойностите са ЕДНАКВИ.
   - JNE (Jump if Not Equal): Прескочи, ако стойностите са РАЗЛИЧНИ.
   - JG (Jump if Greater): Прескочи, ако RAX е ПО-ГОЛЯМО.
   - JL (Jump if Less): Прескочи, ако RAX е ПО-МАЛКО.

3. ПРИМЕР В ЕЗИКА C И СЪОТВЕТСТВИЕ В АСЕМБЛЕР:
   В C пишете:
   if (number == 100) {
       // направи нещо
   }

   В Асемблер това става така:
   CMP RAX, 100
   JE RAVNO_E  ; Ако RAX е 100, отиди на етикет RAVNO_E

4. ГРАНИЧНИ УСЛОВИЯ:
   - Ако забравим да сложим CMP преди условния преход (JE/JG), процесорът ще вземе грешно решение на базата на предишна операция!
   - Логическите проверки изискват регистърът да съдържа валидни данни.
